## 1. テスト方針

### 目的

本テスト仕様書は、営業日報システムの品質を担保するために実施するテストの内容・手順・合否判定基準を定める。

### テストレベル

```
E2Eテスト（画面操作）
        ↑
  結合テスト（API）
        ↑
    単体テスト
```

単体テストで関数・クラス単位の正確性を担保し、結合テストでAPIの入出力・ロール制御を検証する。E2Eテストで実際の画面操作シナリオを確認する。

### テスト記号凡例

| 記号 | 意味 |
|------|------|
| ○ | 正常に動作すること（期待結果と一致） |
| ✕ | エラーになること（期待するエラーが返ること） |
| — | 対象外 |

### 不具合の管理

| 重要度 | 定義 | リリース判定 |
|--------|------|------------|
| Critical | システムが起動しない・データ消失・セキュリティ脆弱性 | 解消するまでリリース不可 |
| High | 主要機能（日報作成・提出・コメント）が動作しない | 解消するまでリリース不可 |
| Medium | 一部機能の動作不良・表示崩れ | 回避策がある場合はリリース可（次リリースで対応） |
| Low | 軽微な表示崩れ・文言の誤り | リリース可（計画的に対応） |
